Syntax

Use either a conventional formula syntax like 
formulas(sos).

	all x (p(x) -> (q(x) | r(x))).
	all x (q(x) -> r(x)).

end_of_list.

formulas(goals).

	all x (p(x) -> r(x)).

end_of_list.
for the Mace4 and Prover9 demonstrators.

or that for the classic demonstrator: 
(a -> b) & a & -b

For conventional formula syntax:

  • Negation symbols are -
    e.g.: (-p)
  • Conjunction symbols are &
    e.g.: (p & q & r)
  • Disjunction symbols are |
    e.g.: (p | q)
  • Implication symbols are ->
    e.g.: (p -> q)
  • Backward implication symbols are <-
    e.g.: (p <- q)
  • Equivalence symbols are <->
    e.g.: (p <-> q)
  • Universal quantification all
    e.g.: (all x all y p(x,y))
  • Existential quantification exists
    e.g.: (exists x exists y p(x,y))

Symbols With Meaning
Several symbols have built-in meaning. These are the equality symbols (=, !=) and logic connectives (-, |, &, ->, <-, <->, all, exists). These symbols can be changed as described in the section Redeclaring Built-in Symbols. (Parentheses, comma, period, and the list construction symbols cannot be redeclared.)

Clauses
The disjunction (OR) symbol is |, and the negation (NOT) symbol is -. The disjunction symbol has higher precedence than the equality symbol, so equations in clauses do not need parentheses. Every clause ends with a period. Examples of clauses follow (Prover9 adds some extra space when printing clauses). 

formulas(sos).
   	p|-q|r.
   	a=b|c!=d.
   	f(x)!=f(y)|x=y.
end_of_list.

For Prover9 formulas, each quantified variable must have its own quantifier; Otter allows quantifiers to be omitted in a sequence of quantified variables with the same quantifier. For example, Otter allows (all x y z p(x,y,z)), and Prover9 requires (all x all y all z p(x,y,z)). more information
